In our previous post, we introduced the concept of rendering patterns. Today, we'll delve into Static Sites, a fundamental approach to web rendering. Static sites are pre-rendered and delivered to the user as simple HTML files, ensuring fast load times and minimal server load.

We think this conversational style might be a bit more fun and easier to follow than a regular article. Hopefully, it makes learning about these rendering patterns less boring and more hands-on!

<Conversation.Thread from="user">
  That sounds impressive 👏. Are there any trade-offs or limitations
  to consider 🤔?
</Conversation.Thread>

<Conversation.Thread from="server">
  <div>
    Full Page Reloads: Navigating via full page reloads can lead to a
    less than optimal user experience due to increased wait times
    between page transitions.
  </div>
  <div>
    Static websites are perfect for content that rarely changes. If
    you need frequent updates, you'll want a dynamic website that's
    easier to modify.
  </div>
  <div>
    Managing content on static websites is more complex than on
    dynamic sites. Any changes require modifying the site's files and
    re-uploading them to the server.
  </div>
  <div>
    Static sites serve the same content to every user. This means you
    can't tailor the experience to individual preferences or
    behaviors, which is possible with dynamic sites.
  </div>
</Conversation.Thread>
